---
title: "Get Threat Model"
api: "GET /api/scans/{id}/threat-model"
description: "Retrieve the AI-generated threat model for a scan"
---

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier (UUID) of the scan
</ParamField>

## Response

Returns a threat model object containing security boundaries, attack surfaces, and data flows:

<ResponseField name="id" type="string">
  Unique identifier (UUID) of the threat model
</ResponseField>

<ResponseField name="scan_id" type="string">
  ID of the scan this threat model belongs to
</ResponseField>

<ResponseField name="repo_id" type="string">
  ID of the repository
</ResponseField>

<ResponseField name="summary" type="string" optional>
  High-level summary of the application's security posture
</ResponseField>

<ResponseField name="boundaries_json" type="object" optional>
  Trust boundaries and security zones within the application
  
  <Expandable title="Boundaries Structure">
    The boundaries object typically contains:
    - **internal**: Components within the trusted boundary
    - **external**: External systems and third-party integrations
    - **user_facing**: User-facing interfaces and endpoints
  </Expandable>
</ResponseField>

<ResponseField name="surfaces_json" type="object" optional>
  Attack surfaces and entry points
  
  <Expandable title="Surfaces Structure">
    The surfaces object typically contains:
    - **api_endpoints**: REST/GraphQL API endpoints
    - **authentication**: Authentication mechanisms
    - **file_uploads**: File upload handlers
    - **database_queries**: Database interaction points
  </Expandable>
</ResponseField>

<ResponseField name="data_flows_json" type="object" optional>
  Data flow diagrams showing how sensitive data moves through the system
  
  <Expandable title="Data Flows Structure">
    The data flows object typically contains:
    - **flows**: Array of data flow objects
    - **sensitive_data**: Types of sensitive data handled
    - **data_stores**: Where data is persisted
  </Expandable>
</ResponseField>

<ResponseField name="model_version" type="integer">
  Version number of the threat modeling algorithm used
</ResponseField>

<ResponseField name="edited_by" type="string" optional>
  User ID who last manually edited the threat model
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp when the threat model was created
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp when the threat model was last updated
</ResponseField>
