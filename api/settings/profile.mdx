## Update Profile

<api>PATCH /api/settings/profile</api>

Update the authenticated user's display name.

### Authentication

Requires authentication via session cookie or bearer token.

### Request Body

<ParamField body="display_name" type="string" required>
  New display name for the user. Must not be empty after trimming whitespace.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the profile update succeeded
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X PATCH https://api.heimdall.dev/api/settings/profile \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"display_name": "Jane Smith"}'
```

```javascript JavaScript
const response = await fetch('https://api.heimdall.dev/api/settings/profile', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    display_name: 'Jane Smith'
  })
});
```

```python Python
import requests

response = requests.patch(
    'https://api.heimdall.dev/api/settings/profile',
    headers={'Authorization': 'Bearer YOUR_TOKEN'},
    json={'display_name': 'Jane Smith'}
)
```
</CodeGroup>

### Response Example

```json
{
  "status": "ok",
  "data": {
    "success": true,
    "message": "Profile updated."
  }
}
```

### Error Responses

<ResponseField name="error" type="object">
  <Expandable title="properties">
    <ResponseField name="code" type="number">
      HTTP status code
    </ResponseField>
    <ResponseField name="message" type="string">
      Error description
    </ResponseField>
  </Expandable>
</ResponseField>

**400 Bad Request**: Display name is empty after trimming

**404 Not Found**: User not found in database

## Change Password

<api>POST /api/settings/change-password</api>

Change the authenticated user's password. Requires the current password for verification.

### Authentication

Requires authentication via session cookie or bearer token.

### Request Body

<ParamField body="current_password" type="string" required>
  The user's current password for verification
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password. Must be at least 8 characters long.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the password change succeeded
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api.heimdall.dev/api/settings/change-password \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "current_password": "old_password",
    "new_password": "new_secure_password"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.heimdall.dev/api/settings/change-password', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    current_password: 'old_password',
    new_password: 'new_secure_password'
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.heimdall.dev/api/settings/change-password',
    headers={'Authorization': 'Bearer YOUR_TOKEN'},
    json={
        'current_password': 'old_password',
        'new_password': 'new_secure_password'
    }
)
```
</CodeGroup>

### Response Example

```json
{
  "status": "ok",
  "data": {
    "success": true,
    "message": "Password changed successfully."
  }
}
```

### Error Responses

**400 Bad Request**: 
- New password is less than 8 characters
- Current password is incorrect

**404 Not Found**: User not found in database

### Security Notes

- Passwords are hashed using Argon2id before storage
- Current password must be verified before allowing change
- No password complexity requirements beyond minimum length

See `src/routes/settings.rs:453` (update_profile) and `src/routes/settings.rs:506` (change_password) for endpoint implementations.
