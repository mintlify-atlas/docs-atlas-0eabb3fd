---
title: "Get Repository"
api: "GET /api/repos/{id}"
description: "Retrieve details of a specific repository by ID"
---

## Authentication

This endpoint does not require authentication in the current implementation, but it's recommended to use authenticated requests for production deployments.

## Path Parameters

<ParamField path="id" type="uuid" required>
  The unique identifier of the repository to retrieve.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates whether the request was successful.
</ResponseField>

<ResponseField name="data" type="object">
  The repository object.
  
  <Expandable title="Repository Object">
    <ResponseField name="id" type="uuid">
      Unique identifier for the repository.
    </ResponseField>
    
    <ResponseField name="user_id" type="uuid">
      ID of the user who created the repository.
    </ResponseField>
    
    <ResponseField name="org_id" type="uuid" optional>
      ID of the organization this repository belongs to, if applicable.
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Name of the repository.
    </ResponseField>
    
    <ResponseField name="source_type" type="string">
      Type of repository source. Possible values:
      - `github` - GitHub repository via OAuth
      - `gitlab` - GitLab repository via OAuth
      - `git_url` - Generic Git repository
      - `zip` - Uploaded ZIP file
    </ResponseField>
    
    <ResponseField name="remote_url" type="string" optional>
      Remote URL of the repository.
    </ResponseField>
    
    <ResponseField name="default_branch" type="string" optional>
      Default branch for scanning.
    </ResponseField>
    
    <ResponseField name="last_commit_sha" type="string" optional>
      SHA of the last commit that was scanned.
    </ResponseField>
    
    <ResponseField name="oauth_connection_id" type="uuid" optional>
      ID of the OAuth connection used for authentication, if applicable.
    </ResponseField>
    
    <ResponseField name="issue_auto_create_enabled" type="boolean">
      Whether automatic issue creation is enabled for this repository.
    </ResponseField>
    
    <ResponseField name="issue_auto_create_min_severity" type="string">
      Minimum severity level for automatic issue creation. Possible values: `critical`, `high`, `medium`, `low`.
    </ResponseField>
    
    <ResponseField name="created_at" type="datetime">
      Timestamp when the repository was created.
    </ResponseField>
    
    <ResponseField name="updated_at" type="datetime">
      Timestamp when the repository was last updated.
    </ResponseField>
    
    <ResponseField name="deleted_at" type="datetime" optional>
      Timestamp when the repository was deleted (soft delete). `null` if not deleted.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET https://api.heimdall.example.com/api/repos/01936d2f-8c4e-7890-b123-456789abcdef \
  -H "Cookie: session=your_session_token"
```

## Error Responses

<ResponseField name="error" type="object">
  Error information when the request fails.
  
  <Expandable title="Error Object">
    <ResponseField name="success" type="boolean">
      Always `false` for error responses.
    </ResponseField>
    
    <ResponseField name="code" type="integer">
      HTTP status code.
    </ResponseField>
    
    <ResponseField name="message" type="string">
      Error message describing what went wrong.
    </ResponseField>
  </Expandable>
</ResponseField>

### 404 Not Found

Returned when the repository with the specified ID does not exist.

```json
{
  "success": false,
  "code": 404,
  "message": "Repo '01936d2f-8c4e-7890-b123-456789abcdef' not found"
}
```

### 500 Internal Server Error

Returned when the server encounters an error retrieving the repository.

```json
{
  "success": false,
  "code": 500,
  "message": "Failed to fetch repo: database connection error"
}
```
